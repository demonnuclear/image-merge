## 1. 项目背景

飞牛服务器上有两个目录存放了大量图片，其中大部分是重复的。
用户需要将两个目录合并为一个，去除重复文件，并且可以自由选择合并方向。

本项目同时也是你（一位有 18 年后端经验的 Java/C# 开发者）学习 Python 的实战练习项目。
代码中会包含极其详细的注释，帮助你理解每一行 Python 代码的含义。

## 2. 功能需求

### FR-01: 双目录配置

-   用户通过 Web 页面配置两个图片目录的路径（目录 A、目录 B）
-   支持配置合并方向：
    -   **A → B**：将 A 合并到 B，A 中不重复的文件复制到 B，A 中已存在于 B 的删除（或移动到回收区）
    -   **B → A**：将 B 合并到 A，B 中不重复的文件复制到 A，B 中已存在于 A 的删除（或移动到回收区）
-   配置项持久化存储（保存到 JSON 文件）

### FR-02: 扫描与分析

-   递归扫描两个目录下的所有图片文件
-   对每张图片计算两个标识：
    -   **SHA256 哈希**：判断文件内容的精确重复（只要文件内容完全一样就是重复）
    -   **感知哈希（pHash）**：判断图片视觉内容的相似性（尺寸不同、有水印、压缩率不同但视觉相同的图片也能识别）
-   分析结果将图片分为以下几类：
    -   **完全重复（SHA256 相同）**：文件内容完全一致
    -   **视觉重复（SHA256 不同但 pHash 相似）**：内容视觉上相同但文件不同（如不同分辨率、不同格式、不同压缩率）
    -   **唯一文件**：仅在一个目录中存在

### FR-03: 分析报告（Web 展示）

-   以 Web 页面展示分析结果报告，包含：
    -   目录 A 和 B 各自的图片总数、总大小
    -   完全重复的文件列表（A ↔ B 重复）
    -   目录内自重复的文件列表（A 内部重复、B 内部重复）
    -   视觉相似的文件列表（SHA256 不同但图片内容相同）
    -   唯一文件列表（仅存在于 A 或 B）
    -   去重后可以释放的磁盘空间预估
    -   合并后的目录预期文件数和大小
-   报告支持可视化展示（表格 + 统计数字 + 进度条）

### FR-04: 合并方案（用户确认）

-   根据扫描结果和合并方向，自动生成合并方案
-   方案中明确列出：
    -   将要被删除/保留的重复文件
    -   将要被复制的唯一文件
    -   操作后的目录结构预览
-   用户查看方案后，必须手动点击「确认合并」按钮才能执行
-   确认后自动执行：复制唯一文件 → 删除源目录的重复文件 → 完成后给出总结

### FR-05: 安全性

-   删除文件前先备份到回收区（在目标目录下创建 `.recycle` 目录）
-   提供「预览模式」：只扫描分析，不执行任何写操作
-   所有操作都有日志记录（操作时间、文件路径、操作类型、结果）

## 4. 系统架构

### 4.1 整体流程

```
用户配置目录路径和合并方向
      │
      ▼
开始扫描 ──→ 计算 SHA256 + pHash
      │
      ▼
生成分析报告（Web 展示）
      │
      ▼
用户查看报告
      │
      ▼
生成合并方案（根据合并方向）
      │
      ▼
用户确认 → 执行合并 → 完成总结
```

### 4.2 模块划分

本项目按功能分为以下模块（每个模块一个 Python 文件）：

| 模块文件 | 功能 | 对应你的 Java/C# 理解 |
|----------|------|----------------------|
| `main.py` | 程序入口，启动 Flask 服务 | 类似 `Main.java` / `Program.cs` |
| `config.py` | 配置的读取和保存 | 类似 `ConfigManager.java` |
| `scanner.py` | 扫描目录、计算哈希 | 类似文件遍历 + MD5 工具类 |
| `analyzer.py` | 分析重复关系、分类 | 类似业务逻辑 Service |
| `reporter.py` | 生成报告数据 | 类似 DTO / ViewModel |
| `merger.py` | 执行合并操作（复制/删除） | 类似文件操作 Service |
| `templates/` | HTML 页面（Jinja2 模板） | 类似 JSP / Razor 视图 |
| `static/` | CSS、JS 等静态资源 | 同 Java/C# Web 项目 |

### 4.3 数据流

```
                 ┌─────────────┐
                 │ config.json │ ← 持久化配置
                 └──────┬──────┘
                        │
  ┌──────────┐    ┌─────┴──────┐    ┌──────────┐
  │ 目录 A/B │───→│  scanner.py │───→│ config.py │
  └──────────┘    └─────┬──────┘    └──────────┘
                        │ 文件列表 + 哈希
                        ▼
                 ┌─────────────┐
                 │ analyzer.py │ ← 分析重复关系
                 └──────┬──────┘
                        │ 分析结果
                        ▼
                 ┌─────────────┐
                 │ reporter.py │ ← 生成报告数据
                 └──────┬──────┘
                        │ 报告 JSON
                        ▼
                 ┌─────────────┐
                 │  Web 页面   │ ← 展示给用户
                 └──────┬──────┘
                        │ 用户确认
                        ▼
                 ┌─────────────┐
                 │  merger.py  │ ← 执行合并
                 └─────────────┘
```

## 5. Web 页面设计

### 5.1 页面结构

| 页面 | 路由 | 功能 |
|------|------|------|
| 首页 | `/` | 配置目录路径和合并方向 + 启动扫描 |
| 报告页 | `/report` | 展示分析报告结果 |
| 方案页 | `/plan` | 展示合并方案 + 确认按钮 |
| 执行页 | `/execute` | 执行合并并展示结果 |
| 日志页 | `/logs` | 查看操作日志 |

### 5.2 页面原型（文字描述）

**首页：**
- 标题：「图片去重合并工具」
- 表单输入：
  - 目录 A 路径（文本框）
  - 目录 B 路径（文本框）
  - 合并方向（单选：A→B / B→A）
  - 预览模式（复选框：仅扫描不操作）
- 按钮：「开始扫描」

**报告页：**
- 统计卡片：A 数量、B 数量、重复数量、唯一数量、可释放空间
- 表格：重复文件列表（显示文件名、大小、SHA256 前 16 位）
- 表格：视觉相似文件列表
- 表格：唯一文件列表
- 按钮：「查看合并方案」

**方案页：**
- 详细操作清单：将要复制/删除的文件列表
- 确认按钮：「确认执行合并」
- 取消按钮：「返回修改」

## 6. 技术要点（给 Java/C# 开发者的说明）

以下是 Python 技术要点，方便你在学习时对照：

| 概念 | Python 实现 | 和你熟悉的 Java/C# 对比 |
|------|------------|----------------------|
| 哈希计算 | `hashlib.sha256()` | ≈ Java `MessageDigest` / C# `SHA256Managed` |
| 文件遍历 | `os.walk()` | ≈ Java `Files.walk()` / C# `Directory.EnumerateFiles()` |
| 图片处理 | `PIL.Image.open()` | ≈ Java `ImageIO` / C# `Image.FromFile()` |
| Web 路由 | `@app.route('/')` | ≈ Java `@RequestMapping` / C# `[Route]` |
| HTML 模板 | Jinja2 `{{ var }}` | ≈ JSP EL `${var}` / Razor `@Model.Name` |
| JSON 处理 | `json.dump/load` | ≈ Jackson / System.Text.Json |
| 异常处理 | `try/except` | ≈ try/catch |
| 路径操作 | `pathlib.Path` | ≈ `java.nio.file.Path` / C# `Path` |

## 7. 部署说明（预研）

在飞牛 OS 上的部署步骤：

1. 通过 SSH 登录飞牛 OS
2. 确保 Python 已安装：`python3 --version`
3. 安装 pip：`sudo apt install python3-pip`
4. 安装依赖：`pip install flask pillow imagehash`
5. 上传项目文件到飞牛 OS
6. 启动：`python3 main.py`
7. 通过浏览器访问 `http://飞牛IP:5000`
